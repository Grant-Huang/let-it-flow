# 决策约束条件模板

模型给建议时必须考虑的硬约束。

## 安全红线（不可越）

- **停线操作**（`edge.pause_line` / 设备 status=down）：必须双确认（governance 规则）
- **工艺参数调整**：必须工艺工程师签 confirm（HITL）
- **报废处置**：必须质量负责人确认

## 资源约束

- **备件库存 = 0** 时：建议换件需先 `equipment.spare_parts` 查件，缺件则建议"临时方案 + 加急采购"
- **产能利用率 > 100%**：建议加单需先评估排产（`schedule.capacity`）
- **库存 < 8h**：建议维持产能需先 `material.inventory` 评估缺料风险

## 证据强度约束（precondition）

诊断类任务的硬前置条件（应用 boot.ts 注册）：
- OEE 诊断：必须先有 `oee.realtime` + `equipment.downtime`
- 质量诊断：必须先有 `quality.defect_rate` + `quality.pareto`
- 能耗诊断：必须先有 `energy.realtime` + `energy.by_process`

证据不足时不要硬给建议，应继续调工具补齐。

## 给建议的执行度评分参考

| executionScore | 含义 | 典型场景 |
|----------------|------|----------|
| >0.8 | 易执行 | 调参数/培训/SOP 更新 |
| 0.5-0.8 | 中等 | 换件/小改造/流程调整 |
| <0.5 | 难执行 | 设备大修/工艺重设计/组织变革 |

# PDCA 阶段判定（给模型的推理脚手架）

模型在分析时，应先判断当前问题处于 PDCA 哪一阶段，再选合适工具。

## 四阶段与工具映射

### P（Plan，规划）
- **特征**：用户问"应该怎么做"/"改善方向"
- **必备证据**：现状基线（OEE/质量/能耗）+ 历史趋势
- **工具优先**：`oee.history` + `oee.trend` + `core.knowledge_base`（查方法论）
- **输出**：改善方向 + 优先级排序

### D（Do，执行）
- **特征**：用户问"现在状况如何"/"是否在执行"
- **必备证据**：实时数据 + 排产达成
- **工具优先**：`oee.realtime` + `schedule.attainment` + `equipment.status`
- **输出**：执行状态快照

### C（Check，检查）
- **特征**：用户问"改善有效吗"/"指标好转了吗"
- **必备证据**：改善前 vs 改善后对比数据
- **工具优先**：`oee.trend` + `oee.by_shift` + `quality.fpy`
- **输出**：变化幅度 + 显著性判断

### A（Act，处置）
- **特征**：用户问"为什么没达标"/"下一步"
- **必备证据**：根因证据 + 损失分解
- **工具优先**：`oee.decompose` + `equipment.downtime` + `quality.pareto` + `process.deviation`
- **输出**：根因链 + 调整建议

## 判定规则（给模型）

读用户意图时，按关键词初判阶段：
- "怎么办/方向/计划" → P
- "现在/正在/实时" → D
- "效果/好转/对比" → C
- "为什么/没达标/根因" → A

模糊时默认 A（最常见场景是诊断问题）。

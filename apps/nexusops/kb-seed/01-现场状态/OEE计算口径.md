# OEE 计算口径

OEE（Overall Equipment Effectiveness，设备综合效率）是衡量产线效率的核心指标。

## 公式

```
OEE = 可用率 × 性能率 × 质量率
```

## 三大分项定义（企业定制版）

### 可用率（Availability）
- **定义**：实际运行时间占计划运行时间的比例
- **公式**：可用率 = （计划时间 − 停机时间）/ 计划时间
- **扣除项**：故障停机、换模、缺料待机、计划保养
- **不扣项**：班次间休息（不计入计划时间）

### 性能率（Performance）
- **定义**：实际产出速度相对理论速度的比例
- **公式**：性能率 = （总产量 × 理论节拍）/ 运行时间
- **扣减项**：小停机（<5min）、降速运行、空转
- **注意**：理论节拍以工艺标准速度为准，非历史最佳速度

### 质量率（Quality）
- **定义**：合格品占全部产出（含报废+返工）的比例
- **公式**：质量率 = 合格品 / 总产出
- **扣减项**：报废、返工、降级品
- **首检+末检**：不计入产出

## 世界级基准

| 等级 | OEE | 说明 |
|------|-----|------|
| 世界级 | ≥85% | 行业标杆 |
| 良好 | 70-85% | 多数成熟企业水平 |
| 改进空间 | 60-70% | 需系统性改善 |
| 不足 | <60% | 急需诊断 |

## 各分项阈值（本企业判定标准）

| 分项 | 世界级 | 合格 | 预警 | 严重不足 |
|------|--------|------|------|----------|
| 可用率 Availability | ≥90% | 85-90% | 75-85% | <75% |
| 性能率 Performance | ≥95% | 90-95% | 80-90% | <80% |
| 质量率 Quality | ≥99% | 95-99% | 90-95% | <90% |
| 综合 OEE | ≥85% | 70-85% | 60-70% | <60% |

> 三项阈值有"短板效应"：任何一项掉到"严重不足"，综合 OEE 都会暴跌。诊断时按"最大损失项"优先。

## 六大损失分类（与三大分项的映射）

| 损失类型 | 归属分项 | 典型原因 |
|----------|----------|----------|
| 故障停机（Breakdown） | 可用率 | 设备故障、突发停机 |
| 换模/换型（Changeover） | 可用率 | 换模超时、换型调试 |
| 小停机/空转（Minor Stop） | 性能率 | 卡料、传感器误触发、临时卡顿 |
| 降速运行（Reduced Speed） | 性能率 | 设备带病、工艺保守限速 |
| 启动损失（Startup Loss） | 性能率 | 开机初期不稳定 |
| 缺陷/报废/返工（Defect） | 质量率 | 工艺不良、参数漂移 |

> 可用率损失还含：缺料待机、计划保养。这些虽非"六大损失"但本企业单列追踪。

## 诊断优先级

OEE 低于目标时，按"最大损失项"优先诊断：
1. 可用率 < 85% → 查 `equipment.downtime`（停机原因帕累托）
2. 性能率 < 90% → 查 `process.deviation` + 小停机（`equipment.minor_stops`）
3. 质量率 < 95% → 查 `quality.pareto` + 工艺参数

> 交叉判定：若多项同时低，按"可用率 > 质量率 > 性能率"顺序攻关（可用率往往是基础，可用率低会连锁影响其他）。

## 关联工具

- `oee.realtime`：实时 OEE + 三项分解
- `oee.decompose`：损失瀑布定位最大损失项
- `equipment.downtime`：可用率损失根因
- `quality.pareto`：质量损失根因

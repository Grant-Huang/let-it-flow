## 一、设备致质量 / 设备致 OEE

### 症状：批量报废 + 加工面损伤
- [主轴轴承异响致批量报废](../../02-改善项目/A3报告库/主轴轴承异响案例.md)
- 诊断模式：`equipment.health` < 0.6 + 振动报警 + 质量缺陷
- 关键工具：`equipment.health` → `equipment.alarm_history` → `quality.pareto`
- **分析方法**：单视角（设备健康+质量关联）

### 症状：可用率损失 + 换模超时
- [换模超时致可用率损失](../../02-改善项目/A3报告库/换模超时致可用率损失案例.md)
- 诊断模式：`oee.decompose` 显示可用率主损 + `schedule.changeover` > 60min
- 关键工具：`oee.decompose` → `schedule.changeover` → `schedule.bottleneck_resource`
- **分析方法**：单视角（换模 SMED）

## 二、工艺致质量

### 症状：尺寸超差 + Cpk 不足
- [Cpk 不足致尺寸超差](../../02-改善项目/A3报告库/Cpk不足致尺寸超差案例.md)
- 诊断模式：`process.deviation` 超规范 + `quality.cp_cpk` < 1.0
- 常见根因：温控失灵 / 液压泄漏 / 传感器漂移
- 关键工具：`quality.cp_cpk` → `quality.spc` → `process.parameters`
- **分析方法**：多视角并行（5Why + 鱼骨图 + FMEA + 交叉印证，置信度 0.9）—— 工艺致质量天然适配多视角
- **关联 skill**：`skill.multi_perspective_rca`

### 症状：缺陷率周期性波动（按班次）
- 诊断模式：`oee.by_shift` 显示班次差异 > 5%
- 常见根因：交接班 SOP 执行差异 / 某班次人员新
- 待补充案例（当前为诊断模式，无完整案例）

## 三、物料致 OEE

### 症状：缺料停线 + 库存不足
- [缺料停线案例](../../02-改善项目/A3报告库/缺料停线案例.md)
- 诊断模式：`material.inventory` < 8h + `material.supply_risk` 高
- 关键工具：`material.wip_level` → `material.supply_risk` → `material.inventory`
- **分析方法**：单视角（物料库存）

### 症状：WIP 堆积 + 物料流堵塞
- [WIP 堆积致物料流堵塞](../../02-改善项目/A3报告库/WIP堆积致物料流堵塞案例.md)
- 诊断模式：`material.wip_level` 超红线但瓶颈后缺料 + VSM 流效率 <5%
- 关键工具：`material.flow` → `schedule.bottleneck_resource` → `schedule.ct_vs_takt`
- **分析方法**：单视角（VSM 物料流）

## 四、能耗异常

### 症状：能耗飙升 + 单耗升高
- [能耗飙升诊断案例](../../02-改善项目/A3报告库/能耗飙升诊断案例.md)
- 诊断模式：`energy.realtime` > 1.2× 基线 + carbonKgPerUnit 升高
- 关键工具：`energy.by_process` → `process.parameters` → `equipment.health`
- **分析方法**：多视角并行（5Why + 鱼骨图 + FMEA + 交叉印证，置信度 0.9）—— 能耗异常是多域交叉型，鱼骨图优势明显
- **关联 skill**：`skill.multi_perspective_rca`

## 复用建议

1. **先匹配症状**：对照"症状"列，找到最接近的历史案例
2. **复用诊断路径**：参考"关键工具"的调用顺序，避免从零摸索
3. **警惕根因差异**：症状相似但根因可能不同，复用时仍需取数验证
4. **沉淀新案例**：每次解决新问题后，按 A3 模板沉淀回本库

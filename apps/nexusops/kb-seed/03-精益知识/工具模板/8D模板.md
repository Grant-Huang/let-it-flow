# 8D（八步问题解决法）报告模板

适用于：重复出现的、跨部门的质量/设备问题。

## 8 个步骤

| D | 步骤 | 关键产出 |
|---|------|----------|
| D1 | 组建团队 | 跨职能成员（工艺+设备+质量+生产） |
| D2 | 描述问题 | 5W2H（何人/何事/何时/何地/为何/如何/多少） |
| D3 | 临时遏制措施 | 止损（如停线/筛选/换料） |
| D4 | 根因分析 | 5Why + 鱼骨图，**至少问到第 3 层根因** |
| D5 | 永久对策选型 | 多方案对比（impact × executionScore） |
| D6 | 实施并验证 | 试点 → 全推 → 数据验证 |
| D7 | 预防再发 | 标准化（改 SOP / 加防错 / 改 PFMEA） |
| D8 | 团队认可 | 沉淀到知识库（如本 vault 的 A3 案例库） |

## 与 NexusOps 的配合

- D2 描述问题：用 `oee.realtime` + `quality.defect_rate` 取数
- D4 根因：用 `quality.root_cause_5m1e` + `process.fmea`
- D5 对策：用 `nexus_advise` 生成多方案 + impact/executionScore
- D8 沉淀：写入本 vault 的 `02-改善项目/A3报告库/`
